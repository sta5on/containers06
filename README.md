# Лабораторная работа №6

## Цель работы

Получить практические навыки управления взаимодействием нескольких контейнеров Docker на примере PHP-приложения, работающего через связку `nginx` и `php-fpm`.

## Задание

Создать PHP-приложение на базе двух контейнеров:

- `nginx`
- `php-fpm`

Использование `docker-compose` для решения лабораторной работы запрещено.

## Описание выполнения работы

### 1. Подготовка структуры проекта

В корне проекта были созданы каталоги:

- `mounts/site` для файлов PHP-сайта
- `nginx` для конфигурации веб-сервера
- `scripts` для удобного запуска и остановки контейнеров

Также был создан файл `.gitignore` со строками:

```gitignore
# Ignore files and directories
mounts/site/*
```

Дополнительно оставлено исключение для `mounts/site/.gitkeep`, чтобы пустая структура каталога сохранялась в репозитории.

### 2. Подготовка PHP-сайта

В директории `mounts/site` размещен тестовый PHP-сайт. Он выводит:

- текущее время сервера;
- версию PHP;
- тип SAPI;
- имя контейнера, в котором обрабатывается PHP-код.

Это позволяет наглядно показать, что страница открывается через `nginx`, а обработка PHP выполняется контейнером `php-fpm`.

### 3. Настройка nginx

В файле `nginx/default.conf` размещена переопределенная конфигурация:

```nginx
server {
    listen 80;
    server_name _;
    root /var/www/html;
    index index.php;
    location / {
        try_files $uri $uri/ /index.php?$args;
    }
    location ~ \.php$ {
        fastcgi_pass backend:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

Конфигурация направляет все PHP-запросы из контейнера `frontend` в контейнер `backend` по имени `backend` и порту `9000`.

### 4. Создание сети Docker

Для связи контейнеров используется пользовательская сеть:

```bash
docker network create internal
```

### 5. Запуск контейнера backend

Контейнер `backend` запускается на образе `php:7.4-fpm`:

```bash
docker run -d \
  --name backend \
  --network internal \
  -v "$(pwd)/mounts/site:/var/www/html" \
  php:7.4-fpm
```

### 6. Запуск контейнера frontend

Контейнер `frontend` запускается на образе `nginx:1.23-alpine`:

```bash
docker run -d \
  --name frontend \
  --network internal \
  -p 80:80 \
  -v "$(pwd)/mounts/site:/var/www/html" \
  -v "$(pwd)/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro" \
  nginx:1.23-alpine
```

### 7. Автоматизированный запуск

Чтобы не вводить команды вручную каждый раз, добавлены скрипты:

```bash
chmod +x scripts/run.sh scripts/stop.sh scripts/status.sh
./scripts/run.sh
```

Для остановки и очистки:

```bash
./scripts/stop.sh
```

Для вывода только нужных контейнеров и сети перед скриншотами:

```bash
./scripts/status.sh
```

### 8. Проверка работы

После запуска контейнеров сайт открывается по адресу:

```text
http://localhost
```

Для дополнительной проверки можно выполнить команды:

```bash
docker ps
docker network inspect internal
curl http://localhost
```

## Ответы на вопросы

### Каким образом в данном примере контейнеры могут взаимодействовать друг с другом?

Контейнеры взаимодействуют через общую пользовательскую сеть `internal`. Контейнер `frontend` принимает HTTP-запросы и передает обработку PHP-файлов в контейнер `backend` по протоколу FastCGI.

### Как видят контейнеры друг друга в рамках сети internal?

Внутри сети Docker контейнеры видят друг друга по именам контейнеров как по DNS-именам. Поэтому в конфигурации `nginx` используется адрес `backend:9000`.

### Почему необходимо было переопределять конфигурацию nginx?

Стандартная конфигурация `nginx` из образа по умолчанию не настроена на передачу PHP-запросов в `php-fpm`. Без переопределения конфигурации `nginx` либо отдавал бы статические файлы, либо показывал стандартную страницу приветствия, но не выполнял бы PHP-код.

## Выводы

В ходе лабораторной работы было создано PHP-приложение, разделенное на два контейнера: `frontend` на базе `nginx` и `backend` на базе `php-fpm`. Контейнеры были объединены в пользовательскую сеть `internal`, благодаря чему `nginx` смог передавать PHP-запросы в `php-fpm`. Работа показала, как организовать взаимодействие нескольких контейнеров Docker без использования `docker-compose`.
